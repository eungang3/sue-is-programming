## 0. HTTP 프로토콜이란?
- HyperText Transfer Protocol의 약자
- 클라이언트와 서버가 주고받는 메시지의 규칙

<br/>

### 1) request
- 클라이언트가 서버에게 보내는 메시지
> GET / HTTP/1.1
> Host: www.test.com
>> GET은 request의 종류를 나타냄
>> /는 웹사이트의 홈페이지를 뜻함
>> HTTP/1.1은 HTTP 프로토콜의 버전 정보
>> ...는 정보를 더 보낼수도 있음을 뜻함

<br/>

### 2) response
- request를 받은 서버가 클라이언트에게 보내는 메시지

> HTTP/1.1 200 OK
> Content-Type: text/html
> ...
>> 200은 문제 없다는 상태 코드
